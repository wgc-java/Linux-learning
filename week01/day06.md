## Linux 权限体系全景总结（用户/所有者/组/家目录/root/操作顺序）
# 1. 五大核心概念关系图
root（超级管理员 - 上帝模式）
│
├──────────────────────────────────┐
│                                  │
用户（User）                          组（Group）
├── 登录系统的账号                      ├── 用户的集合
├── 如：wgc、zhangsan                  └── 如：dev、docker
│                                  │
└── 与"所有者"是同一事物的两个视角      └── 与"所属组"是同一事物的两个视角
│                                  │
└──────────┬───────────────────────┘
│
文件/目录（File/Dir）
┌─────────┴─────────┐
▼                   ▼
所有者（Owner）        所属组（Group）
├── 从文件视角看       ├── 从文件视角看
└── 就是某个具体用户    └── 就是某个具体组
│
│
家目录（Home）
└── /home/用户名
├── 特殊目录，默认权限700
└── 所有者=该用户，所属组=该用户主组
# 2、关键关系（一对多）
关系	说明	示例
用户 : 文件	一个用户可拥有无数个文件	`wgc` 拥有 `/home/wgc/*` 下所有文件
用户 : 组	一个用户可在多个组（主组+附加组）	`wgc` 主组是 `wgc`，附加组是 `dev`、`docker`
组 : 用户	一个组可有多个用户	`dev` 组包含 `wgc`、`zhangsan`、`lisi`
# 3、权限检查优先级（访问文件时的判断流程）
-rwxr-x---  1  wgc  dev  file.txt
  ↑↑↑  ↑↑↑  ↑↑↑   │    │
  │││  │││  │││   │    └── 所属组：dev
  │││  │││  │││   └─────── 所有者：wgc
  │││  │││  ││└─────────── 其他人权限
  │││  │││  └──────────── 组权限
  │││  │└──────────────── 组写权限
  │││  └───────────────── 组执行权限
  ││└──────────────────── 组读权限
  │└───────────────────── 所有者写权限
  └────────────────────── 所有者读权
# 判断顺序（Linux 按此顺序匹配，一旦匹配成功就不再往下检查）：
1.
是不是所有者？ → 用前3位  rwx （你是房主，给你最高权限）
2.
是不是所属组成员？ → 用中间3位  rwx （你是部门同事，给你部门权限）
3.
都不是 → 用后3位  rwx （你是外人，给你公共权限）
关键：即使你在  dev  组，但如果你就是文件的所有者，只看所有者权限，不看组权限！

# 4、文件vs目录：权限含义完全不同（重点）
权限	对文件（File）	对目录（Dir）
r (读)	能看内容 `cat file`	能列出内容 `ls dir`
w (写)	能改内容 `echo > file`	能增删文件 `touch/rm`
x (执行)	能运行 `./file`	能进入 `cd dir`
# 5、家的特殊逻辑
drwx------  3  wgc  wgc  /home/wgc
  ↑         ↑   │    │        │
  │         │   │    └────── 所属组（默认与用户名同名）
  │         │   └─────────── 所有者（该用户）
  │         └─────────────── 硬链接数#  
  └───────────────────────── 权限700（只有所有者能进）
默认权限 700（ rwx------ ）：连同组人也进不来，保护隐私
要共享给组：先  chgrp  改组，再  chmod 750  给组  rx  权限（能进门、能看，不能改）
 # 6、标准操作流程（sop)
场景：让dev组的同事能进入你的家目录查看文件，但不能改
 步骤1：查看现状（组是wgc，权限700）
ls -ld /home/wgc
 drwx------ 5 wgc wgc /home/wgc

 步骤2：改组归属（换部门管）
sudo chgrp dev /home/wgc
 现在：drwx------ 5 wgc dev /home/wgc
       组变了，但组权限还是---（进不来）

 步骤3：改组权限（发卡：能进门+能看，不能改）
chmod 750 /home/wgc
 现在：drwxr-x--- 5 wgc dev /home/wgc
           ↑↑↑
           组有 r-x（读+执行=能cd进来+能ls）
# 最后的快速记忆表
问题	答案
一个用户能有多个组吗？	✅ 能，一个主组+多个附加组
一个组能有多个用户吗？	✅ 能
一个用户能拥有多个文件吗？	✅ 能，无数个
文件能被"进入"吗？	❌ 不能，只有目录能被 `cd` 进入（需要 x 权限）
所有者是用户吗？	✅ 是，"所有者"是从文件视角看的用户
root 受权限限制吗？	❌ 不受，root 无视 rwx 位
先改组还是先改权限？	先 `chgrp`（定归属），再 `chmod`（定权限）
家的默认权限？	700（`rwx------`），连组都进不来
